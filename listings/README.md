# Build listing archive

**Why:** memory/stack dumps from hang/crash forensics are useless without the
symbol table of the EXACT build that produced them. The working-tree
`chess-engine.lst` gets overwritten by every rebuild — archive here at every
flash.

**Convention:** after flashing firmware, copy the listing here as
`chess-engine-<short-commit-hash>.lst`. If the tree was dirty at flash time,
use a descriptive suffix and rename once committed
(e.g. `chess-engine-diag-0717-uncommitted.lst` -> `chess-engine-4f34ae2.lst` (done 7/17)`).

**Dump-set -> listing map:**
- `hang-0717-memdump-*.log` (Caro-Bf5 d2 freeze, 2026-07-17)
  -> `chess-engine-8c7208d.lst` (regenerated byte-exact from source via git
  worktree — build is deterministic, so a lost listing is recoverable for any
  committed build; UNCOMMITTED builds are NOT recoverable: archive at flash time).
- `crash-v3-memdump-*.log` + `elph-debug-crash-v3-0703.log` (v3 crash, 2026-07-03)
  -> king-safety-v3 branch build (5a4eaad era); regenerate via worktree if the
  August investigation resumes.
- Older `bp*`/`bpt1*`/`brkn*` dumps (May-June freeze hunt) -> pre-freeze-fix
  builds; commits identifiable from git log dates if ever needed.

Verified return addresses in the 7/17 hang stack (against 8c7208d listing):
$2AB3 = root CALL NEGAMAX (SEARCH_POSITION), $21BB = recursive CALL NEGAMAX
($21B8), $24F5 = CALL QUIESCENCE_SEARCH ($24F2).
